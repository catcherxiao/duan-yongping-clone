# 雪球抓取说明

## 当前结论

段永平雪球账号这条线可以做，但不适合匿名硬爬。

我在 2026-04-07 的实测里确认了两点：

- 段永平雪球主页可见用户 ID：`1247347556`
- 匿名访问主页和时间线接口都会被雪球的阿里云 WAF 挡住

所以更稳的办法是：

- 你本地浏览器登录雪球
- 导出完整 Cookie 请求头
- 用项目里的脚本本地归档
- 或者直接启动带远程调试的 Chrome，让 agent 在你登录后抓截图、页面文本和 cookies

脚本位置：

- [scripts/fetch_xueqiu_timeline.py](/Users/catcher_agent/Desktop/codex/段永平skill/scripts/fetch_xueqiu_timeline.py)
- [scripts/chrome_cdp_capture.mjs](/Users/catcher_agent/Desktop/codex/段永平skill/scripts/chrome_cdp_capture.mjs)

## 怎么拿 Cookie

最简单的方法：

1. 在浏览器里登录雪球。
2. 打开段永平主页或任意雪球页面。
3. 打开开发者工具。
4. 找到一个发往 `xueqiu.com` 的请求。
5. 复制完整的 `Cookie` 请求头。

不要只复制单个 cookie 键值，最好复制整行 Cookie。

---

## 浏览器登录后让 agent 接手

如果你想让我在你手动登录后继续操作，可以走这条链路：

1. 启动带远程调试的 Chrome。
2. 你在浏览器里手动登录雪球。
3. 我用 CDP 脚本抓当前标签页截图、文本快照或 cookies。

常用命令：

```bash
node scripts/chrome_cdp_capture.mjs list --port 9222
node scripts/chrome_cdp_capture.mjs screenshot --port 9222 --match xueqiu
node scripts/chrome_cdp_capture.mjs snapshot --port 9222 --match xueqiu
node scripts/chrome_cdp_capture.mjs cookies --port 9222 --match xueqiu
```

这样做的好处是：

- 先由你自己完成真实登录
- 我不需要你手动导出每一步页面内容
- 能直接保存截图和当前页面文本，便于后续整理

## 本地运行示例

### 方式 1：直接用环境变量

```bash
export XUEQIU_COOKIE='xq_a_token=...; xqat=...; u=...'
python3 scripts/fetch_xueqiu_timeline.py --user-id 1247347556 --max-pages 200
```

### 方式 2：放到本地文件

```bash
mkdir -p .secrets
printf '%s\n' 'xq_a_token=...; xqat=...; u=...' > .secrets/xueqiu.cookie
python3 scripts/fetch_xueqiu_timeline.py \
  --user-id 1247347556 \
  --cookie-file .secrets/xueqiu.cookie \
  --max-pages 200
```

## 关于“全部问答”

脚本当前做的是：

- 先归档用户时间线全部原始内容
- 再按关键字筛一份 `qa-candidates.ndjson`

这已经足够做第一轮语料沉淀，但它还不等于严格意义上的“全部问答”。

原因是：

- 雪球问答很多散在评论、回复、互动链里
- 不同接口返回结构可能不一样
- 有些内容需要更深的登录态和更细的接口扩展

所以更现实的工作流是：

1. 先把时间线全量存下来。
2. 再从高价值条目往下追评论和回复。
3. 最后把真正有价值的 Q&A 沉淀成摘录和主题卡片。
