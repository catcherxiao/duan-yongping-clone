# Local Source Materials

这个目录给你本地维护语料用，不要求发布到公开仓库。

建议结构：

- `speeches/`
  - 放讲话原文、访谈逐字稿、股东会问答、播客转录
- `books/`
  - 放你个人合法持有并仅供本地研究使用的书籍、摘录、OCR 文本
- `notes/`
  - 放你自己整理的摘录、索引、时间线和主题卡片

建议命名方式：

- `YYYY-MM-DD-title.md`
- `YYYY-book-title.md`
- `topic-name-notes.md`

使用建议：

- 原始材料尽量保留出处、日期、链接、版本说明
- 长材料不要一次性全塞进 skill，上下文里按需读取片段
- 书籍全文如果有版权风险，尽量只保留本地，不要公开提交到 GitHub
- 更推荐在 `notes/` 里做结构化摘录，而不是把所有原文都直接喂给模型

当前已经整理好的本地笔记示例：

- `notes/大道-目录与章节索引.md`
- `notes/大道-高频主题卡片.md`
- `notes/大道-问答表达模式.md`
- `notes/大道-高价值问答案例块.md`

推荐工作流：

- 先把原始 PDF、讲话、访谈放到 `books/` 或 `speeches/`
- 再用 `notes/` 做章节索引、主题卡片、时间线、表达模式
- skill 默认优先读取 `notes/`，只有必要时再回到原始材料
- 如果要直接产出回答，可在 `notes/` 里额外维护“案例块”文件，把高价值问答压成可复用的回答骨架
- 如果要驱动本地网页 demo，可以再把成熟的案例块同步到 `demo/data/case-blocks.json`
