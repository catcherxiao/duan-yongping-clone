# 泡泡玛特问答自审记录

日期：2026-04-09

## 自审题目

`对泡泡玛特这家公司你怎么看？`

## 旧版主要问题

- 会混入明显偏旧的 `2022`、`2023` 论据
- 有时会把检索结果写成“搜索摘要拼贴”，不像老段平时说话
- 局部会把来源名直接带进回答
- 有些论据虽然新，但不够关键，容易跑偏成细枝末节

## 这次调整

- 检索 query 改成优先 `最新 + 财报/业绩会/管理层/风险 + 2026 2025`
- 结果筛选改成按发布时间、年份、领域关键词、站点质量一起打分
- 默认优先最近 `12-18` 个月的论据，旧年份信息降权
- 过滤低质量摘要，比如 `We cannot provide a description for this page right now`
- 给模型更强约束：
  - 不要写成研报
  - 先判断，再给 1 到 2 个关键新论据
  - 不在正文展示来源名
  - 句子更短，更像聊天

## 当前结果检查

- `来源：`：未出现
- `2022年`：未出现
- `2023年`：未出现
- `2025年`：出现
- `2026年`：出现

## 当前效果判断

- “像老段”的程度：明显比之前好
- “论据新鲜度”：明显比之前好
- “信息密度”：够用
- “最近事件感”：已经有，但后面还可以继续压缩成更直接的说法

## 剩余可继续优化的点

- 个别回答还是偏长，可以再收短一点
- 某些场景仍可能挑到“新闻评论”而不是“财报/业绩会/官方口径”
- 后面可以继续加一层“站点白名单 + 年份阈值”，让近期官方和业绩会信息更稳
